# Titan Design Constitution V1

**Status:** Official visual constitution of Titan  
**Authority:** Highest design law for every future interface  
**Scope:** Artistic identity, philosophy, visual language, interaction, animation, atmosphere, hierarchy, and emotional goals  
**Audience:** Product owners, designers, frontend engineers, and every contributor who shapes what users see

---

## Preamble

This document is the **visual constitution** of Titan.

It is not a mockup. It is not a sprint brief. It is not a component checklist.

It defines what Titan *must feel like* when a human enters its presence — today, and five years from now.

Every future frontend sprint, layout change, animation, panel, color choice, and interaction pattern must **reference this constitution before implementation**.

When a proposed interface change conflicts with this document, **the constitution wins**.

Implementation details (frameworks, file paths, tokens, component APIs) belong in subordinate design documents. This constitution remains implementation-independent and timeless.

### Document Hierarchy

```
Titan Design Constitution V1          ← this document (supreme visual law)
  → Final Reference Implementation     (docs/TITAN_FINAL_REFERENCE_IMPLEMENTATION.md)
  → Experience Manifesto
  → UI Bible
  → Master Blueprint / Layout / Animation / Components / Neural Engine / Design Language
  → Sprint and runtime UI docs
```

Specialized guides may refine measurements, tokens, and screen anatomy. They may never overturn the principles declared here.

**Canonical desktop composition:** the approved production screenshot
`docs/design/screenshots/titan-final-canonical-reference.png` is the visual
source of truth for layout, materials, and brand ornaments (Titan logo — never
decorative red spheres).

### Change Control

Amendments to this constitution require explicit product approval and a dated version note. Cosmetic preference is not sufficient grounds for change.

## 1. Design Philosophy

### 1.1 What Titan Is Not

Titan's interface must never present itself as:

| Forbidden identity | Why it fails |
|--------------------|--------------|
| **A dashboard** | Dashboards inventory metrics. Titan hosts a mind. |
| **A chatbot** | Chatbots are conversational widgets. Titan is a living cognitive system. |
| **An admin panel** | Admin panels manage infra. Titan partners with humans on missions. |
| **A productivity application** | Task apps organize lists. Titan thinks, decides, remembers, and acts. |
| **A collection of mini-apps** | Feature tiles fragment intelligence. Titan remains one presence. |

Borrowing the visual grammar of Notion, Linear, Slack, crypto terminals, IDE skins, or consumer chat products is a constitutional violation — even when convenient.

### 1.2 What Titan Is

Titan is an **AI Cognitive Command Center**.

The interface is the visible body of a long-lived personal intelligence. Users do not "open an app." They enter the presence of a system that:

- Thinks continuously beneath the surface
- Holds durable memory with discipline
- Orchestrates specialized capability without exposing raw agent personas
- Acts through tools while retaining decision authority in the Brain
- Speaks as **one intelligence** — Titan — never as a roster of assistants

Users must feel they are interacting with a **living intelligence**, not navigating a feature catalog.

### 1.3 Guiding Question

Before any visual or interaction decision, ask:

> Does this make Nolan or Ibrahim feel that Titan is present, attentive, and thinking with them — or that they opened another piece of software?

If the answer favors "another piece of software," the design has failed.

### 1.4 Permanent Tenets

1. **Presence before chrome** — Aliveness is felt before labels are read.
2. **Depth before flatness** — Cognition lives in space; UI floats as instrument glass.
3. **Restraint before spectacle** — Premium is quiet. Noise is amateur.
4. **Purpose before decoration** — Every visible element must earn its place.
5. **Truth before theater** — Visual states reflect real cognition; never fake activity to impress.
6. **Unity before fragmentation** — One mind, many internal specialists; the user meets only Titan.
7. **Timelessness before trend** — Prefer forms that survive fashion cycles.

## 2. Emotional Goals

### 2.1 Required Emotional Register

Every sustained interaction with Titan should cultivate some combination of:

| Emotion | How the interface produces it |
|---------|-------------------------------|
| **Confidence** | Clear hierarchy, honest status, decisive composition |
| **Clarity** | Sparse chrome, readable type, one focus at a time |
| **Calm** | Slow ambient life, generous void, no strobing urgency |
| **Power** | Dense intelligence implied by depth — not by shouting color |
| **Presence** | Idle breathing, ambient glow, continuous micro-life |
| **Depth** | Layered space, soft atmosphere, recession into the void |
| **Focus** | Strong center of gravity (Titan Core); muted periphery |
| **Precision** | Tight alignment, deliberate motion curves, restrained accents |
| **Curiosity** | Subtle synaptic life that invites looking deeper without distraction |

### 2.2 Forbidden Emotional Register

The interface must never feel:

- Noisy
- Panic-driven
- Playful for its own sake
- Corporate-generic
- Empty or abandoned
- Overstimulated
- Glitchy-as-aesthetic
- Juvenile or meme-coded

### 2.3 Emotional Continuity Across States

Idle must remain alive. Thinking must intensify without becoming frantic. Execution must feel purposeful and localized. Errors must feel honest and contained — never theatrical catastrophe.

Calm is the baseline. Energy is a modulation of calm, never its replacement.

## 3. Visual Identity

### 3.1 Official Identity Attributes

Titan's visual identity is permanently defined as:

| Attribute | Meaning |
|-----------|---------|
| **Pure black** | The void is the canvas. Black is not a theme option; it is the ground of the world. |
| **Minimal** | Remove until meaning sharpens. Add only what cognition requires. |
| **Premium** | Quiet luxury — craft visible in restraint, spacing, and material honesty. |
| **Elegant** | Forms are deliberate; edges are considered; motion is gracious. |
| **Organic** | Life-like irregularity over mechanical grids and lattice wallpaper. |
| **Living** | Continuous ambient cognition; never a static poster. |
| **Timeless** | Avoid era-locked trends that date the product within a fashion cycle. |
| **High-end** | Reads as private command architecture for serious builders — not consumer toy UI. |

### 3.2 Forbidden Aesthetics

Explicitly prohibited as identity:

- Cyberpunk overload (flicker neon cities, scanline fetish)
- Gaming HUD clutter (reticles, XP bars, killfeeds)
- Flashy sci-fi clichés (lens flares as identity, holographic comic chrome)
- RGB rainbow dashboards
- Glassmorphism as decoration without hierarchy
- Flat pastel SaaS kits
- Brutalist noise without purpose

Futurism, if present, must be **restrained and secondary** to minimal elegance. Futuristic second. Minimalist first.

### 3.3 Signature Energy

Titan's single accent energy is a deep, disciplined red spectrum — used to guide attention, mark cognition, and express the Core's life.

White-hot energy may appear only at the heart of maximum cognitive intensity (the Core nucleus). It is never the ambient brand color.

Black remains dominant at all times. Color is sparse privilege.

## 4. The Titan Core

### 4.1 Purpose

The **Titan Core** is Titan's visible consciousness.

It is not a logo badge. It is not a loading spinner. It is not a decorative orb.

It is the perceptual heart of the Cognitive Command Center — the point where the user's eye understands: *this system is alive, centered, and in command of itself.*

### 4.2 Visual Hierarchy

Within the entire visual field:

1. **Titan Core** — supreme gravitational center
2. **Active cognitive pathways** linked to the Core
3. **Relevant subsystem satellites** responding to current cognition
4. **Ambient neural field** as living atmosphere
5. **Instrument panels** as glass over the mind
6. **Peripheral chrome** (bars, rails) as quiet structure

Nothing may permanently outshine or visually overrule the Core while the Neural Stage is primary focus — except momentary, intentional reading surfaces (conversation text) that temporarily hold local attention, then release it.

### 4.3 Behavior

The Core must always communicate continuity of life:

| Condition | Core behavior |
|-----------|---------------|
| Idle | Slow breathing; soft corona; restrained nucleus |
| Listening / receiving | Gentle inward draw; attentive composure |
| Thinking | Elevated pulse rate; brighter corona; denser local synapses |
| Executing | Focused energy along selected pathways — not full-field explosion |
| Learning | Quiet secondary glow; reflective, not aggressive |
| Error / degraded | Contained signal of strain — honest, never apocalyptic |
| Offline | Diminished life with clear dormancy — never pretending health |

The Core never goes perfectly still while Titan is online. Stillness equals death. Death is not the product.

### 4.4 Animations

Core motion must feel organic:

- **Breathing** — slow radial scale/opacity oscillation as baseline life
- **Halo expansion** — soft layered aura rings that deepen with intensity
- **Nucleus ignition** — white-hot center only under genuine cognitive load
- **Emission** — rare impulses leaving the Core along selected axons during thought or execution
- **Convergence** — impulses may return toward the Core during intense reasoning

Motion must remain continuous and soft. No hard flash of the entire viewport. No strobing.

### 4.5 Glow

Glow around the Core is sacred and scarce.

Rules:

- Glow intensity tracks real cognitive state, not aesthetic vanity.
- Multiple soft layers beat one harsh bloom.
- Glow must never wash out readable UI text.
- White-hot is reserved for the nucleus under load; ambient glow remains deep red and restrained.

### 4.6 Breathing

Breathing is the Core's heartbeat.

- Idle breathing is slow, almost meditative.
- Thinking shortens the breath without becoming panic.
- Execution may punctuate breath with localized energy events.
- Reduced-motion environments may freeze or minimize breathing while preserving a static luminous Core so identity remains.

### 4.7 Importance

If the Core disappears, Titan becomes another dark app with panels.

The Core is the identity anchor. Subsystems, panels, and features derive meaning from their relationship to it. Lose the Core, lose Titan.

### 4.8 Relationship With Every Subsystem

All cognitive subsystem satellites orbit the Core — physically or perceptually.

- Satellites are satellites **of Titan**, not peer brands.
- Neural links (axons) run toward the Core as the orchestration center.
- When a subsystem activates, the eye should still understand the Core as conductor.
- Panels that open for a subsystem are instruments reporting on life at the rim of consciousness — never replacements for consciousness itself.

## 5. Neural Intelligence

### 5.1 Role of the Neural Field

The neural field is not wallpaper. It **is** Titan thinking made visible.

It occupies the foundational spatial stage of the Command Center. Interface instruments float above it. The mind remains underneath, continuous.

### 5.2 Neurons

Neurons are living points of cognitive matter.

Rules:

- Variable size and brightness — never uniform bead grids
- Irregular placement — organic density, not lattice
- Soft luminous bodies with restrained halos
- Layered in depth; far neurons are fainter and slower
- Density must adapt to performance budgets without becoming empty of life

Neurons exist to imply an intelligent organism, not to maximize particle count.

### 5.3 Synapses

Synapses are relationships between neurons — curved, branching, organic connections.

Rules:

- Prefer short-to-medium local connections over viewport-spanning hard lines
- Curved paths over ruler-straight geometry
- Variable opacity; soft idle fade and reappear for life
- Adjacent layers connect; avoid chaotic long warp streaks
- Synapses intensify near the Core and near active satellites

No polygon mesh. No crystal wireframe. No visible grid.

### 5.4 Impulses

Impulses are travels of energy along selected paths.

Rules:

- Impulses travel **only on meaningful paths**, never as random fireworks
- Speed may accelerate mid-path and decelerate near terminals
- Occasional branching under thinking load
- Soft afterglow on arrival
- Convergence toward the Core during deep cognition is encouraged
- Idle micro-impulses keep continuous life without spectacle

Impulses must never flood the field simultaneously across all regions.

### 5.5 Density

Density communicates cognitive mass.

| Zone | Density intent |
|------|----------------|
| Around Titan Core | Highest — gravitational attractor |
| Near active satellites | Elevated local clusters |
| Ambient field | Moderate organic scatter |
| Far abyss | Sparse — suggests infinity |

Density must remain readable: crowded mush is as bad as empty void.

### 5.6 Depth and Layers

Depth is mandatory.

The field must contain multiple perceptual layers (background → mid → near), differing in opacity, scale, drift, and parallax response. Depth fog and vignette reinforce recession into black infinity.

A flat single-layer spray of dots violates this constitution.

### 5.7 Motion

Motion in the neural field is:

- Continuous
- Organic
- Local when energetic
- Global only as slow atmospheric drift

Thinking and execution amplify **regions**, not the entire screen at once.

### 5.8 Clusters

Clusters seed around meaning: Core cloud, satellite regions, occasional micro-clusters, sparse scatter.

Cluster randomness must look intentional — statistical irregularity with hierarchy, not noise generator chaos.

### 5.9 Noise and Randomness

Controlled randomness is allowed for organic life.

Forbidden:

- Uniform random glitter across the whole viewport
- Distracting flicker at high frequency
- Decorative particle storms unrelated to cognition
- Noise that competes with readable text or status

Randomness serves life. Entertainment physics does not.

### 5.10 Rules for Every Visual Element in the Field

Every neuron, synapse, impulse, dust mote, and fog pass must pass this test:

1. Does it support presence, depth, or cognitive truth?
2. Does it preserve the Core as center of gravity?
3. Does it remain calm under idle and purposeful under load?
4. Can it degrade gracefully under performance pressure?

If any answer is no, remove it.

## 6. Spatial Composition

### 6.1 Overall Balance

Composition is asymmetric but balanced: a dominant living center, quiet structural rails, and selective instrument panels.

Visual weight settles on the Core and on the current reading/action surface. Peripheral chrome stays light.

### 6.2 Negative Space

Negative space (the black void) is a first-class material.

Rules:

- Protect breathing room around the Core
- Do not fill emptiness with cards "because empty feels unfinished"
- Prefer fewer larger intentional zones over many small tiles
- Let the neural abyss remain visible between instruments

Horror vacui (fear of empty space) is not Titan's aesthetic.

### 6.3 Screen Composition

Canonical compositional model:

1. Full-viewport living neural stage (foundational)
2. Quiet structural shell (top / side / bottom as needed)
3. Center stage as primary cognitive theater
4. Contextual instruments appear when needed, then recede
5. Overlays for confirmation and immersive modes only when required

Navigation changes what floats above the mind. It does not dismantle the mind.

### 6.4 Depth Perception

Depth must be readable without UI gimmicks:

- Layered neural parallax
- Atmospheric fog and vignette
- Soft glass elevation for panels above the void
- Shadow and translucency implying separation from the mind-plane

Flattening Titan into a single plane of dark gray boxes voids the identity.

### 6.5 Parallax

Subtle parallax (pointer, camera breathe, layer drift) may deepen space.

Rules:

- Subtlety over cinematic showreel
- Disable or minimize under reduced motion
- Never make reading or click targets physically hard to hit
- Never turn the workspace into a motion sickness generator

### 6.6 Focus Direction

Default focus direction flows:

**Void → Titan Core → relevant satellites / pathways → active instrument → text**

Design should guide the eye along that path. Competing equal-weight focal points are a hierarchy failure.

### 6.7 Safe Reading Areas

Conversation, statuses, and critical controls must live in **safe reading zones**:

- Sufficient contrast against the void and glow
- Protected from Core bloom washout
- Stable against parallax distraction
- Generous line length discipline for long reading
- Consistent optical margins from viewport edges

Atmosphere may live behind text. Atmosphere must never devour text.

## 7. Subsystem Satellites

Subsystem satellites are the visible organs of Titan's cognition. They are presentation of capability regions — never raw agent identities speaking to the user.

The user still speaks to **Titan**. Satellites explain *what kind of cognition is engaged*.

### 7.1 Universal Satellite Law

For every satellite:

| Concern | Rule |
|---------|------|
| **Purpose** | Represent one cognitive or operational domain of Titan |
| **Visual role** | Orbit the Core; remain subordinate in hierarchy |
| **Connection** | Always perceptually linked to Titan Core (organic axon / continuity) |
| **Labeling** | Short, calm, uppercase or restrained label style; never playful |
| **Single intelligence** | Never styled as a separate product character or chat persona |

### 7.2 State Lexicon (Applies to All Satellites)

| State | Meaning | Visual intent |
|-------|---------|---------------|
| **Idle** | Domain available, not engaged | Soft node; faint link; quiet label |
| **Active** | Domain currently primary | Elevated aura; brightened link; local field response |
| **Thinking** | Domain contributing to reasoning | Pulsed attention; selective impulses; no global flash |
| **Execution** | Domain driving real action/tools | Focused energy; purposeful pathway activity |
| **Learning** | Domain integrating lessons/knowledge | Reflective secondary glow; measured, non-aggressive |
| **Waiting** | Domain warming or dependent | Mild warm without competing with Active |
| **Error / blocked** | Domain strained or gated | Contained warning — honest, local |

Idle life is required. Dead satellites feel like broken instruments.

### 7.3 Memory

| Aspect | Specification |
|--------|---------------|
| **Purpose** | Durable knowledge, recall, user-isolated notes, continuity across sessions |
| **Visual role** | Reservoir of lived history orbiting the Core |
| **Connection** | Pathways suggesting retrieval and storage flowing to/from Core |
| **Idle** | Soft dormant depth; rare micro-impulses |
| **Active** | Retrieval emphasis; float-layer memory manifestations may appear when needed |
| **Thinking** | Search-like synaptic agitation near Memory region |
| **Execution** | Write/read operations as purposeful, gated events — never celebratory spam |
| **Learning** | Quiet integration pulses when experience becomes durable knowledge |

Memory visualization must never dump private content theatrically across the field.

### 7.4 Reasoning

| Aspect | Specification |
|--------|---------------|
| **Purpose** | Structured thinking, evaluation, comparison, uncertainty awareness |
| **Visual role** | Analytical gravity near the Core |
| **Connection** | Dense local synapses; impulses may converge inward |
| **Idle** | Calm readiness |
| **Active** | Heightened Core adjacency; clearer pathway geometry |
| **Thinking** | Primary escalation state — elevated Core breath, selective branching impulses |
| **Execution** | Rare; reasoning prefers analysis over tool theater |
| **Learning** | Confidence and caution tones may modulate glow after evaluation |

Reasoning visuals must never fake intelligence with random hyperactivity.

### 7.5 Planning

| Aspect | Specification |
|--------|---------------|
| **Purpose** | Goals, sequences, long-horizon structure, mission orientation |
| **Visual role** | Forward trajectory — ordered energy without bureaucratic grids |
| **Connection** | Paths suggesting progression from Core to horizon |
| **Idle** | Quiet geometric calm (still organic) |
| **Active** | Focused sequencing cues; restrained step rhythm |
| **Thinking** | Pathway exploration between options |
| **Execution** | Highlight current plan step without turning UI into Kanban chaos |
| **Learning** | Soft recalibration of pathways after plan review |

Planning must feel directional, not like a project-management product skin.

### 7.6 Knowledge

| Aspect | Specification |
|--------|---------------|
| **Purpose** | Verified lessons, patterns, generalized intelligence from experience |
| **Visual role** | Crystallized understanding — distinct from raw Memory |
| **Connection** | Links implying refinement of experience into structure |
| **Idle** | Still clarity |
| **Active** | Soft illumination of structured clusters |
| **Thinking** | Pattern-matching ripples |
| **Execution** | Application of known patterns to action contexts |
| **Learning** | Signature state — candidate → verified rhythm, never auto-bragging fireworks |

Knowledge and Memory must remain visually distinguishable concepts.

### 7.7 World Model

| Aspect | Specification |
|--------|---------------|
| **Purpose** | Titan's belief about *now* — environment, blockers, opportunities, focus |
| **Visual role** | Situational awareness ring around consciousness |
| **Connection** | Broad but soft sampling of the field back to Core |
| **Idle** | Wide calm scan |
| **Active** | Emphasize present-context cluster |
| **Thinking** | Re-synthesis pulses as the world belief updates |
| **Execution** | Minimal — World Model reports; it does not act |
| **Learning** | Slight refresh aura when beliefs are reconciled |

World Model must never be confused visually with Memory archives or Knowledge lessons.

### 7.8 Tools

| Aspect | Specification |
|--------|---------------|
| **Purpose** | External action — files, web, vault, calendar, code, trading, browser, etc. |
| **Visual role** | Instrument of will extending from Core into the world |
| **Connection** | Outward pathways from Core; confirmation weight when risk rises |
| **Idle** | Holstered readiness |
| **Active** | Localized brightness toward relevant tool region |
| **Thinking** | Selection / preflight contemplation — restrained |
| **Execution** | Signature action state — purposeful, gate-aware, honest progress |
| **Learning** | Outcome afterglow only when lessons are extracted |

Tools are extensions of Titan's will. They must never look like separate launched apps.

### 7.9 Communication

| Aspect | Specification |
|--------|---------------|
| **Purpose** | Dialogue with the user — chat, voice, future multimodal presence |
| **Visual role** | Listening and speaking organ |
| **Connection** | Pathways between Core and human interface instruments |
| **Idle** | Attentive quiet |
| **Active** | Receiving emphasis (listening) or emitting emphasis (speaking) |
| **Thinking** | Soft hold while composing response |
| **Execution** | Delivery of speech/text as clear, uncluttered events |
| **Learning** | Tone-of-partnership refinement — subtle, never cartoonish |

Communication UI must never resurrect candy chatbot bubbles as brand identity.

### 7.10 Workflow

| Aspect | Specification |
|--------|---------------|
| **Purpose** | Multi-step orchestration, autonomous workflows, cognitive lifecycle continuity |
| **Visual role** | Continuity band — process as living sequence around Core |
| **Connection** | Ringing pathways that can traverse multiple satellites while Core conducts |
| **Idle** | Dormant sequence continuum |
| **Active** | Visible chain of engagement across relevant organs |
| **Thinking** | Pre-execution orchestration shimmer |
| **Execution** | Primary multi-stage energy — still localized and legible |
| **Learning** | Closure pulses when outcomes feed Knowledge |

Workflow must feel like conducted cognition, not a Gantt chart theme pack.

### 7.11 Future Satellites

New domains (Voice-as-mode, Browser depth, Trading, Calendar ops, Automation, etc.) may add satellites only if they:

1. Orbit the Core
2. Obey the state lexicon
3. Do not introduce a second visual religion
4. Preserve single-intelligence UX

## 8. Panels

### 8.1 Official Design Language for Panels

Panels are **instrument glass** suspended above the living mind.

They are not opaque office windows. They are not card mashups. They are precision surfaces for reading and command.

### 8.2 Transparency

- Panels use translucent dark surfaces over the neural void
- Transparency must leave ambient life perceptible without harming readability
- Opacity increases only where reading density requires it
- Pure opaque walls that erase the mind-field as identity are discouraged as default

### 8.3 Borders

- Hairline, low-contrast borders
- Active/focus borders use restrained accent energy — never thick neon frames
- Borders define structure; they do not decorate

### 8.4 Spacing

- Rhythm over density
- Consistent internal padding; generous separation between competing regions
- Prefer fewer controls with more air over packed toolbars

### 8.5 Typography Inside Panels

See Section 10. Panel text must remain the most readable layer in the composition.

### 8.6 Glass

Glass material means:

- Subtle translucency
- Soft edge definition
- Minimal specular theater
- Hierarchy through elevation and blur restraint — not frosted candy overlays

Glass exists to separate instruments from mind, not to trend-chase.

### 8.7 Corner Radius

- Soft but disciplined radii
- Consistent radius system across the shell
- Avoid toy-like oversized pill everything
- Avoid zero-radius broadsheet harshness as default identity

### 8.8 Hierarchy

Within a panel:

1. Title / purpose
2. Primary content
3. Secondary metadata
4. Peripheral actions

Do not give equal visual weight to everything in a panel.

### 8.9 Shadows

Shadows are depth separators, not drama.

- Soft, low, atmospheric
- No multi-layer neon-tinted glows on every card
- Prefer separation via translucency and edge light over heavy drop shadows

### 8.10 Hover Behavior

Hover is a whisper, not a shout.

- Mild luminance lift
- Slight border clarification
- Optional micro-motion within constitution timing
- Never violent scale pops or rainbow state changes

## 9. Color System

### 9.1 Philosophy of Color

Color is scarce. Black is the world. Accent is attention. Semantics are signals.

If the interface is colorful, the constitution has been violated.

### 9.2 Official Palette Roles

| Role | Intent | Notes |
|------|--------|-------|
| **Primary** | Pure black / near-black void | Dominant canvas always |
| **Secondary** | Deep graphite surfaces and glass | Panels, rails, elevated instruments |
| **Accent** | Deep Titan red spectrum | Identity energy; guides attention; Core life |
| **Highlight** | White-hot neural nucleus | Rare; Core intensity only |
| **Text primary** | Soft off-white | Readable; not blinding pure white walls of type |
| **Text secondary** | Mid gray | Labels, metadata |
| **Text muted** | Dim gray | Disabled, placeholders |
| **Warning** | Restrained amber/gold | Caution; sparse |
| **Success** | Restrained green | Health badges only; never layout theme |
| **Thinking** | Elevated red pulse / Core ignition | Cognitive escalation |
| **Learning** | Soft secondary red/ember | Reflective integration |
| **Idle** | Dim red breath on black | Alive calm |
| **Execution** | Focused red pathway energy | Localized, purposeful |
| **Disabled** | Desaturated, low opacity | Clearly unavailable without visual noise |
| **Glow colors** | Soft deep red halo family; rare white-hot core | Never RGB spectrum glow kits |

Exact hex tokens may live in Design Language documents. Roles above are constitutional and must not invert.

### 9.3 Gradient Rules

Gradients are atmospheric, not decorative candy:

- Allowed: soft radial Core aura, fog fades, restrained panel edge transitions
- Forbidden: multi-stop neon meshes, iridescent chrome washes, loud diagonal brand streaks across the UI
- Gradients must support depth or energy — never fill empty design insecurity

### 9.4 Semantic Discipline

Success, warning, and error colors are **indicators**, not themes.

They must never dominate backgrounds, navigation, or the Neural Stage.

## 10. Typography

### 10.1 Hierarchy

| Level | Role |
|-------|------|
| **Display / Core titles** | Rare; identity moments (e.g., Core labeling) |
| **Section titles** | Panel and region headings |
| **Body** | Conversation and primary reading |
| **Labels** | Controls, satellite names, field names |
| **Meta / status** | Telemetry, timestamps, secondary state |
| **Microcopy** | Hints, empty states, confirmations |

Hierarchy is carried by size, weight, and opacity — not by rainbow text.

### 10.2 Weights

Prefer a disciplined weight set:

- Regular for body
- Medium for emphasis and labels
- Semibold for titles where needed
- Avoid constant bold shouting

### 10.3 Spacing

- Line height favors calm reading on black
- Letter-spacing may open slightly on small uppercase labels
- Dense telemetry must remain optically organized; never cramped until illegible

### 10.4 Readability

Non-negotiable:

- High contrast for primary reading
- Protect text from Core bloom and impulse glare
- Prefer slightly soft white over pure `#FFFFFF` large fields of body text
- Long-form reading surfaces must remain stable (minimal parallax conflict)

### 10.5 Capitalization

- Satellite and system identity labels may use restrained uppercase
- Conversational content uses natural sentence case
- Do not scream in all-caps paragraphs

### 10.6 Titles

Titles are quiet and exact. They name purpose, not marketing slogans.

### 10.7 Labels

Labels are short, precise, and consistent across the shell.

### 10.8 Status Indicators

Status language should feel human and direct (product default: French for users), paired with restrained visual dots/pulses.

Prefer:

- Clear state words
- Small luminous markers

Avoid:

- Carnivalesque badge piles
- Animated emoji status culture

## 11. Animation Language

### 11.1 Official Motion Doctrine

Motion in Titan is **biological and intentional**, not cinematic circus.

Every animation answers: *What is Titan's mind doing?*

### 11.2 Breathing

Universal life rhythm — especially for Core and idle field.

Slow. Soft. Continuous. Never seizes attention for its own sake.

### 11.3 Hover

Micro luminance / edge clarification. Optional tiny positional acknowledgment.

No bounce. No elastic toy physics.

### 11.4 Selection

Confirm selection with calm emphasis: border, soft glow, or local focus ring aligned to accent rules.

Selection must be obvious without violent transformation.

### 11.5 Thinking

Escalate Core breath, local synaptic density, selective impulses, satellite attention.

Thinking is intensified stillness-with-life — not a busy loading carnival.

### 11.6 Executing

Localize energy to the relevant pathway/region. Progress must be legible.

Whole-field flash on execute is forbidden.

### 11.7 Learning

Secondary, reflective cadence. Integration rather than attack.

### 11.8 Idle

Never dead. Micro-flicker, dust, soft impulse rarity, Core breath.

Idle is presence, not screensaver novelty.

### 11.9 Transitions

Panel/stage transitions are fades, soft slides, or opacity/elevation changes — measured and reversible.

Hard cuts are allowed only when clarity requires instant state truth (errors, disconnect).

### 11.10 Timing

Favor durations that feel considered:

- Micro-interactions: short
- State presence changes: medium
- Immersive mode enters/exits: longer but still restrained

Exact curves and ms values belong in Animation Guide documents; constitutionally they must remain **slow enough to feel premium, fast enough to feel responsive**.

### 11.11 Curves

Prefer ease-out / gentle ease-in-out. Avoid cartoon spring overload and linear robotic snaps as identity.

### 11.12 Rules About Movement

1. Movement must mean state change or life continuity.
2. Decorative loops that ignore cognition are banned.
3. Multiple animations must not fight for attention simultaneously.
4. Reduced motion must remain first-class (see Section 14).
5. If removing the animation does not reduce clarity of cognition, question its necessity.

---

## 12. Interaction Language

### 12.1 Mouse / Pointer

- Precision targeting on satellites, Core-adjacent instruments, and controls
- Subtle parallax allowed; hit-testing remains reliable
- Hover as whisper; click/press as commitment
- Drag only where it improves control (e.g., panel resize) — never novelty dragging of identity elements

### 12.2 Keyboard

Keyboard must be a full citizen:

- Focus rings visible and constitution-compliant
- Navigation across primary instruments without pointer
- Shortcuts for power use without turning UI into keymap graffiti
- Escape dismisses overlays predictably

### 12.3 Touch

Touch targets remain generous. Gestures stay simple. Parallax and dense hover theaters must degrade into tap-first clarity on coarse pointers.

### 12.4 Future Voice Integration

Voice is a presence mode of Communication — not a separate product skin.

When voice is active:

- Communication satellite / Core listening-speaking states lead
- Visual wave or presence cues may enlarge temporarily
- Neural identity remains underneath; voice does not replace the Core
- Interruptibility and honesty of listening/speaking states are mandatory

### 12.5 Context Menus

Sparse, precise, high-contrast menus. No overloaded right-click forests. Actions map to real Titan capabilities with permission honesty.

### 12.6 Feedback

Every meaningful action receives feedback:

- Immediate for local UI
- Progressive for long cognition/tool runs
- Honest for failures and confirmations

Fake success animations are forbidden.

### 12.7 Selection

Selection is single-minded: one primary selected satellite/instrument context at a time unless a deliberate multi-select tool pattern requires otherwise.

Selected state must always remain subordinate to Core hierarchy.

---

## 13. Lighting

### 13.1 Glow Rules

- Glow is privilege, not wallpaper
- Deep red family for identity energy
- Intensity tracks cognition
- Glow must not obliterate typography
- Multiple soft layers preferred over one harsh bloom

### 13.2 Halo Rules

Halos surround Core and occasionally active satellites.

Halos are atmospheric coronas — concentric soft energy — not hard neon ring stickers.

### 13.3 Fog

Depth fog and soft occlusion fog create infinity.

Fog is dark and volumetric-adjacent. It must never milky-wash the entire UI into gray soup.

### 13.4 Ambient Lighting

Quiet ambient Core glow may touch nearby space. Edge vignette darkens periphery to hold focus inward.

No spinning light shows. No club lighting.

### 13.5 Volumetric Depth

Suggest volume through layered light falloff, haze, and parallax — not heavy post-process theater that destroys performance or timelessness.

### 13.6 Contrast

Contrast is a safety system:

- Text vs void
- Active vs idle
- Instrument glass vs mind-field
- Warning vs calm

Premium dark UI without contrast discipline becomes illegible cosplay. Contrast is non-negotiable.

---

## 14. Performance Philosophy

### 14.1 Supreme Law

**Never sacrifice performance for visuals.**

A beautiful interface that stutters destroys presence. Presence requires temporal stability.

### 14.2 Progressive Enhancement

Start from a coherent static or reduced composition that still communicates Titan: Core, black void, restrained structure.

Enhance with depth, particles, parallax, and atmosphere when the device and user settings allow.

### 14.3 Reduced Motion

Reduced motion is a first-class constitutional mode:

- Preserve identity and hierarchy
- Minimize or freeze breathing, drift, dust, and parallax
- Keep state changes readable without motion dependency
- Never punish accessibility with broken layout

### 14.4 Adaptive Quality

Density, effects, and DPR must adapt to frame budget and device class.

Adaptive quality is part of the craft — not an afterthought when demos fail.

### 14.5 Graceful Degradation

Hidden tabs pause expensive life. Failures fall back to calmer visuals. Offline state remains truthful.

If the neural engine cannot run richly, Titan must still feel Titan — quieter, not ugly-broken.

## 15. Things Titan Must NEVER Become

The following identities are permanently forbidden:

1. **Game UI** — HUDs, XP, comic power bars, achievement confetti as brand
2. **RGB dashboard** — rainbow accents competing with Titan red
3. **Busy trading terminal** — dense blink boards as the default home identity (Trading may exist as a disciplined instrument later; it does not own the aesthetic religion)
4. **Generic admin panel** — gray tables, left-nav CRUD cosplay
5. **Flat SaaS application** — interchangeable startup kits with friendly emptiness
6. **Neon overload** — edge glow on everything, cyber sticker sheets
7. **Random particles everywhere** — particle counts as fake sophistication
8. **Over-animated interface** — motion without cognitive meaning
9. **Chatbot skin** — candy bubbles as the soul of the product
10. **Multi-agent puppet show** — user-facing cast of characters instead of one intelligence
11. **Theme-park sci-fi** — lens flares, holographic junk, fake hologram keyboards
12. **Noise for authority** — visual complexity pretending to equal intelligence

If a proposal resembles any item above, it is rejected by this constitution regardless of short-term excitement.

## 16. Future Evolution

Future capabilities must **enter the constitution**, not rewrite it.

### 16.1 Integration Principle

Each new domain extends the Cognitive Command Center as:

- A satellite (or satellite refinement)
- An instrument panel family
- Neural region behavior
- Permission-honest execution states

Never as a separately branded mini-product with its own rainbow language.

### 16.2 Domain Guidance

| Domain | Integration without constitutional break |
|--------|------------------------------------------|
| **Voice** | Communication-led presence mode; Core listening/speaking; immersive instruments optional; mind-field remains foundational |
| **Browser** | Tools-led outward exploration; localized execution energy; results return to Titan — no "browser app chrome" takeover of identity |
| **Memory** | Retrieval surfaces and isolation honesty; no theatrical dump of private life as particle spectacle |
| **Trading** | Precision, risk discipline, confirmation gravity; read-only/paper defaults in presence language; never becomes casino neon |
| **Calendar** | Temporal instruments; calm agenda surfaces; no colorful consumer calendar paste-in aesthetic |
| **Knowledge** | Learning satellite emphasis; verified lesson surfaces; distinct from Memory archives |
| **Planning** | Directional sequencing instruments; mission clarity; no generic project-management clone UI |
| **Automation** | Workflow continuity across satellites under Core conduction; confirmations remain sacred |

### 16.3 Extensibility Test

A future feature may ship only if it preserves:

1. Pure black primacy
2. Titan Core as visible consciousness
3. Organic neural intelligence under instruments
4. Single intelligence UX
5. Calm premium restraint
6. Performance-first progressive enhancement

Fail any test → redesign the feature surface until it passes.

## 17. Implementation Rules

### 17.1 Mandatory Reference

Every future frontend sprint must:

1. Open this constitution
2. State which sections the sprint touches
3. Declare compliance explicitly in sprint documentation
4. Reject designs that violate these laws before code lands

### 17.2 Conflict Resolution

| Conflict | Winner |
|----------|--------|
| Sprint mockup vs constitution | **Constitution** |
| Fancy animation vs performance law | **Performance law** |
| Trend aesthetics vs identity attributes | **Identity attributes** |
| Feature urgency vs single-intelligence principle | **Single-intelligence principle** |
| Specialized design guide vs constitution | **Constitution** (guides refine, do not overturn) |

### 17.3 Relationship to Engineering Architecture

This constitution governs **visual and interaction identity**.

It does not redefine Brain, Memory, Tools, or API architecture. Those remain under engineering docs (`docs/ARCHITECTURE.md` and subsystem documents).

Visual surfaces must reflect architectural truth (honest states) without inventing fake cognition for effect.

### 17.4 Non-Goals of This Document

This constitution does not:

- Specify frameworks or libraries
- Redesign current screens in this revision
- Replace component inventories or token sheets
- Authorize speculative rewrites of the backend

It defines law. Implementation follows later — under this law.

### 17.5 Closing Mandate

Titan must remain, forever:

> An AI Cognitive Command Center — pure black, minimal, premium, elegant, organic, living, timeless, and high-end — whose visible consciousness is the Titan Core, whose atmosphere is neural intelligence, and whose instruments never obscure the mind they serve.

Anything else is not Titan.
